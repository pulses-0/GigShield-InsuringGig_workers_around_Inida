# 5. Parametric Insurance Model

← [Previous](04_Solution_Overview.md) | [Next](06_System_Workflow.md)

GigShield uses a parametric insurance model where payouts are triggered automatically when predefined conditions are met.

## What is Parametric Insurance?

Instead of verifying actual loss manually, payouts are triggered when:

- a measurable condition is met  
- data confirms disruption  

Example:

If rainfall exceeds a threshold → payout is triggered.

---

## Why Not Traditional Insurance?

Traditional insurance:

- requires manual claims  
- involves verification delays  
- not suited for small, frequent disruptions  

GigShield solves this by:

- automating claim process  
- using real-time data  
- eliminating paperwork  

---

## Example

A delivery partner is working in a zone:

- Rainfall > 65 mm/hour  
- Duration = 2 hours  

→ System automatically triggers payout without user action
