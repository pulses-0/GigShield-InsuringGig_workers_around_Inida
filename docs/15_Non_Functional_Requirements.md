# 15. Non-Functional Requirements

Non-functional requirements define the quality attributes of the GigShield system.

---

## 15.1 Performance

- trigger detection within 60 seconds  
- payout processing within minutes  

---

## 15.2 Scalability

- supports thousands of concurrent users  
- horizontally scalable backend  

---

## 15.3 Reliability

- fallback mechanisms for API failures  
- minimal downtime  

---

## 15.4 Security

- secure authentication (JWT)  
- encrypted data transmission  

---

## 15.5 Usability

- simple user interface  
- minimal user interaction required  

---

## 15.6 Maintainability

- modular architecture  
- clean separation of components  

---

## 15.7 Availability

- cloud deployment  
- high uptime  

---

## 15.8 Importance

These requirements ensure that the system is not only functional but also:

- efficient  
- reliable  
- scalable  

---
